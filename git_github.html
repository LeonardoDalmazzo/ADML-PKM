Configuração & Segurança (Windows)

Configuração Inicial

Ao pegar uma máquina nova, defina o autor dos commits.

# 1. Definir identidade (Global)
git config --global user.name "Seu Nome"
git config --global user.email "email@ex.com"

# 2. Verificar configurações existentes
git config --list

# 3. Ver onde as configs estão salvas
git config --list --show-origin

Limpeza (Sair do PC)

Vai devolver o computador? Apague seus dados para ninguém "commitar" por você.

# 1. Remover nome e email do Git
git config --global --unset user.name
git config --global --unset user.email
Crítico: O Git no Windows guarda sua senha/token separadamente. Para remover o acesso total:
  1. Abra o menu Iniciar e digite "Gerenciador de Credenciais".
  2. Clique em Credenciais do Windows.
  3. Procure por git:https://github.com e clique em Remover.

Status dos Arquivos (Significados)

  • U / ?? → Untracked (Arquivo novo, não rastreado)
  • A → Added (Adicionado ao stage/index)
  • M → Modified (Modificado)
  • D → Deleted (Excluído)
  • R → Renamed (Renomeado)
  • C → Copied (Copiado)

Conventional Commits

  • feat Nova funcionalidade.
    Ex: "feat: add login support"
  • fix Correção de bug.
    Ex: "fix: header aligment on mobile"
  • docs Apenas documentação.
  • style Formatação (espaços, ;), sem mudar lógica.
  • refactor Refatoração de código.
  • chore Manutenção, configs de build, deps.
Breaking Change (feat!): Use ! para indicar mudanças que quebram compatibilidade.
Ex: feat!: drop support for API v1

Fluxo de Trabalho Padrão

Bash
# 1. Garantir que a main está atualizada
git checkout main
git pull origin main

# 2. Criar branch para a tarefa
git checkout -b feature/nova-funcionalidade

# 3. Codar e adicionar mudanças
git add .

# 4. Commit (Conventional Commits)
git commit -m "feat: adiciona carrossel de imagens"

# 5. Enviar para o GitHub
git push origin feature/nova-funcionalidade

# ... Abrir Pull Request (PR), Revisar e Merge ...

# 6. Limpeza local (após merge aprovado)
git checkout main
git pull origin main
git branch -d feature/nova-funcionalidade

WIP: Trocar de PC (Savepoint)

Use isso quando precisar parar de trabalhar no PC 1 e continuar no PC 2 sem finalizar a tarefa (mantendo arquivos como Modified/Untracked).

No PC 1 (Origem)

# 1. Cria branch temporária de backup
git checkout -b backup-temporario

# 2. Salva TUDO (mesmo incompleto)
git add .
git commit -m "WIP: backup para troca de pc"

# 3. Envia para nuvem
git push origin backup-temporario

No PC 2 (Destino)

# 1. Baixa as novidades
git fetch origin

# 2. Entra na branch de backup
git checkout backup-temporario

# 3. A MÁGICA (Desfaz o commit, mantém arquivos)
git reset --soft HEAD~1
Resultado: O commit de backup some, e seus arquivos voltam a ficar verdes (Staged). Agora você pode continuar codando como se nunca tivesse trocado de PC!

SOS: Recuperação & Emergência

1. Existe no Remoto

Você apagou a branch local, mas já tinha dado push (ela existe no GitHub).

  1. Atualize seu Git:
    git fetch --all
  2. Verifique se ela aparece (vermelha):
    git branch -a
  3. Recupere conectando:
    git checkout -b nome origin/nome

2. Sumiu de tudo (Reflog)

Você apagou local e remoto. O Git local ainda lembra.

  1. Veja o histórico de ações:
    git reflog
  2. Ache o hash (ex: e9404ab).
  3. Ressuscite a branch:
    git checkout -b nome [hash]

Checkout Bloqueado

Erro: "Your local changes to... would be overwritten".

  1. Guarde na gaveta temporária:
    git stash
  2. Faça o checkout/pull que queria.
  3. (Opcional) Devolva as mudanças:
    git stash pop

Correções e Desfazendo

Desfazer 'git add'

git restore --staged arquivo.txt
git restore --staged .

Editar último commit

Altera a mensagem ou adiciona arquivos esquecidos ao último commit (sem criar um novo).

git add arquivo-esquecido.txt
git commit --amend

Commits antigos

(antes do push):

git rebase -i HEAD~3   # últimos 3 commits
# troque 'pick' por 'reword' nos commits desejados

Depois do push (⚠️ use com cuidado):

git commit --amend
git push --force

Gerenciamento de Branches

Deletar

Local (após merge):

git branch -d nome-branch

Local (forçado/sem merge):

git branch -D nome-branch

Remota (GitHub):

git push origin --delete nome-branch

Renomear

Local:

git branch -m novo-nome

Você pode renomear uma branch local sem precisar entrar nela.

git branch -m nome-antigo novo-nome-da-branch

Atualizar Remoto (passo a passo):

  1. git branch -m antigo novo
  2. git push origin -u novo
  3. git push origin --delete antigo

Git no VS Code

  • CTRL + SHIFT + G Abrir Source Control
  • CTRL + SHIFT + G G Commit Rápido (Staged)
  • CTRL + SHIFT + G P Push para remoto
  • CTRL + SHIFT + G U Pull (Atualizar)

Resumo de Comandos

Comando Ação
git init Inicia repositório na pasta atual
git clone [url] Baixa um repositório remoto
git status Mostra arquivos alterados/staged
git reflog Histórico de todas ações locais (Recuperação)
git stash Guarda alterações temporariamente
git diff Mostra diferenças (não staged)
git commit -m "msg" Salva mudanças no histórico
git push Envia commits para o servidor
git pull Baixa e mescla atualizações
git branch Lista branches locais
git checkout [branch] Troca de branch
git merge [branch] Funde a branch especificada na atual
git log --oneline Histórico de commits simplificado
