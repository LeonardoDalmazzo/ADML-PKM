Cliente e Servidor

A internet é baseada em uma conversa entre duas partes:

Cliente (Client)

É o seu navegador (Chrome, Edge). Ele pede informações.

Servidor (Server)

É um supercomputador conectado 24h na internet. Ele guarda os arquivos e responde aos pedidos.

Analogia do Restaurante: Você (Cliente) olha o menu e faz o pedido. O Garçom (HTTP) leva o pedido para a Cozinha (Servidor). A Cozinha prepara o prato (Site) e o Garçom traz de volta para você.

DNS (Domain Name System)

Computadores não entendem nomes como google.com, eles só entendem números (IPs), como 142.250.190.46.

O DNS é a "agenda telefônica" da internet. Ele traduz o nome que você digitou para o IP do servidor correto.

  • 1. Você digita adml.com.
  • 2. O navegador pergunta ao DNS: "Qual o IP desse site?".
  • 3. O DNS responde: "É o IP 192.0.2.1".
  • 4. O navegador conecta nesse IP.

HTTP e HTTPS

HTTP (HyperText Transfer Protocol) é a língua que o Cliente e o Servidor usam para conversar.

Método Ação
GET "Me dá esse site/arquivo" (Entrar numa página).
POST "Toma esses dados" (Fazer login, enviar formulário).
HTTPS: O 'S' é de Secure. Significa que a conversa entre você e o servidor é criptografada. Ninguém no meio do caminho pode ler seus dados.

O que o Servidor devolve?

A "Tríade" do Frontend

  • HTML: A estrutura e conteúdo (O esqueleto).
  • CSS: O estilo e beleza (A roupa).
  • JavaScript: A interatividade e lógica (O cérebro).

O Navegador

O trabalho do navegador é receber esses códigos e "desenhá-los" (renderizar) na tela para que você veja o site pronto, e não um monte de texto.
