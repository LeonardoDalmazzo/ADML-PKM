A Filosofia React

O React não é apenas uma biblioteca para criar interfaces, é uma nova forma de pensar. Em vez de manipular o DOM manualmente (como no jQuery ou Vanilla JS), você define estados.

Regra de Ouro: A interface (UI) é uma função do seu Estado (State).
UI = f(state)

Virtual DOM & Reconciliação

Manipular o DOM real é lento. O React resolve isso criando uma cópia leve na memória: o Virtual DOM.

Como funciona?

  • 1. O estado muda.
  • 2. O React cria uma nova árvore Virtual DOM.
  • 3. Ele compara a nova árvore com a antiga (Diffing).
  • 4. Ele atualiza no DOM Real apenas o que mudou.

Vantagem

Se você alterar apenas um texto em uma lista de 1000 itens, o React não redesenha a lista toda. Ele troca cirurgicamente apenas aquele texto. Isso garante altíssima performance.

JSX: HTML dentro do JS?

JSX (JavaScript XML) parece HTML, mas é açúcar sintático para criar objetos JavaScript. Todo o site é construído combinando pequenos blocos chamados Componentes.

MeuComponente.jsx
function BoasVindas() {
    const nome = "Leonardo";
    
    // Isso parece HTML, mas é JSX!
    return (
        <div className="card">
            <h1>Olá, {nome}!</h1>
            <p>Bem-vindo ao React.</p>
        </div>
    );
}
Note que usamos className em vez de class, pois "class" é uma palavra reservada no JavaScript.

Hooks: useState

Hooks são funções que permitem "ligar" recursos do React (como estado) em componentes funcionais. O mais comum é o useState.

Contador.jsx
import { useState } from 'react';

function Contador() {
    // [valorAtual, funcaoParaMudar]
    const [count, setCount] = useState(0);

    return (
        <button onClick={() => setCount(count + 1)}>
            Cliques: {count}
        </button>
    );
}

Sempre que setCount é chamado, o React sabe que o estado mudou e renderiza o componente novamente com o novo valor.

Props vs State

Props (Propriedades)

São dados passados de pai para filho. São leitura apenas (imutáveis).

Analogia: Como os ingredientes que você entrega para um chef. O chef não muda o ingrediente que recebeu, ele o usa.

State (Estado)

São dados gerenciados dentro do componente. Podem mudar ao longo do tempo.

Analogia: O estado de cozimento do prato dentro da panela. Ele muda enquanto o chef trabalha.
